# Subsystem Overview

## Purpose
PBProjects provides build-time configuration headers that declare compile-time feature flags, library availability, and platform-specific paths for the Aleph One game engine. These headers control which optional subsystems, external dependencies, and platform features are compiled into the executable.

## Key Files
| File | Role |
|------|------|
| config.h | Auto-generated configuration header from autoconf declaring compile-time feature flags and library availability |
| confpaths.h | Configuration header defining platform-specific resource directory paths (currently inactive) |

## Core Responsibilities
- Declare feature detection flags for optional libraries (Boost, SDL, OpenGL, audio codecs, compression)
- Check availability of standard POSIX and C library headers
- Identify platform and version information for conditional compilation
- Control compilation of optional subsystems (CURL, miniupnpc, zlib, libpng)
- Define platform-specific resource file paths (intended; currently disabled)
- Configure runtime resource lookup behavior (LUA_USE_MKSTEMP)

## Key Interfaces & Data Flow
- **Exposes:** Preprocessor constants and conditional feature flags via `#include "config.h"` throughout codebase
- **Consumes:** Generated from autoconf build system (`config.h.in`, `configure.ac`)
- **Dependency declarations:** SDL (image, net, ttf), Boost, OpenGL, zlib, libpng, libyuv, CURL, miniupnpc, POSIX headers (unistd.h, sys/types.h, sys/stat.h, pwd.h, inttypes.h, stdint.h)

## Notable Implementation Details
- `config.h` is auto-generated by autoconf; hand-editing is not recommended
- `confpaths.h` is currently commented out, suggesting the resource path definition was either superseded by runtime discovery or moved to another configuration file
- The macOS path `AlephOneSDL.app/Contents/Resources/DataFiles/` indicates explicit cross-platform build system configuration for app bundle layouts
- `LUA_USE_MKSTEMP` configuration suggests integration with the Lua scripting subsystem for runtime resource management
